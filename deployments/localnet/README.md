# Localnet Deployment Manifests

`pnpm localnet:up` writes `latest.json` in this directory after starting Anvil if needed and deploying the LB v2.2 local stack. It creates one real pool:

- `WETH/USDC` is the transaction, market-activity, indexer, and frontend reference pool. It is funded across on-chain bins around an approximately $2,000 active price.

The deployment runs the WETH/USDC seeder automatically. To add the same distribution to an already-running local deployment, run `pnpm localnet:seed:liquidity`. Re-running the seeder adds liquidity to the existing curve; it does not replace or fabricate indexed data.

Generated JSON manifests are ignored by git because local addresses depend on the local chain state. Operators should not hand-edit `latest.json`; re-run `pnpm localnet:deploy` with the intended environment variables instead.

## Stable Shape

The manifest shape is stable for Wave 1 consumers:

- `contracts`: deployed LBFactory, LBPair implementation, LBRouter, and LBQuoter.
- `ownership`: fee recipient, initial owner, and LBFactory owner.
- `endpoints`: local consumer URLs for RPC, indexer, API, and optionally token list.
- `tokens`: local WNATIVE and mock ERC20 addresses.
- `factoryPreset`: the initial bin-step preset.
- `seededPools`: smoke-test pool addresses and pair parameters.
- `constructorArgs`: constructor values used for the local core deployment.
- `smoke`: liquidity and swap amounts emitted by the deploy smoke.

`startBlock` records the block where the deploy script began. It is usually `0` on a fresh Anvil chain and can be higher when deploying against an already-running RPC. The schema accepts any positive local chain ID; the default helper starts Anvil with `31337`.

## Field Sources

| Manifest field | Source |
| --- | --- |
| `schemaVersion`, `environment`, `sourceJoeV2Commit`, `factoryPreset.*` | Generated by the deploy script. |
| `endpoints.rpcUrl`, `endpoints.indexerUrl`, and `endpoints.apiUrl` | Generated from local defaults or hand-entered `LOCALNET_*_URL` env vars. |
| `endpoints.tokenListUrl` | Hand-entered through `LOCALNET_TOKEN_LIST_URL`; omitted when unset. |
| `chainId`, `startBlock`, deployed `contracts.*`, deployed `tokens.*`, seeded pool addresses, and `smoke.swapAmountOut` | Chain-derived from the target RPC and executed transactions. |
| `deployer`, `ownership.*`, and owner/fee constructor args | Derived from `LOCALNET_PRIVATE_KEY` or the default Anvil key. |
| Legacy router/factory constructor args | Generated zero addresses for the Wave 1 local-only V2.2 path. |
| `smoke.liquidityAmountX`, `smoke.liquidityAmountY`, and `smoke.swapAmountIn` | Generated script constants used to seed and smoke-test the WETH/USDC pool. |

Localnet manifests are meant for SDK examples, frontend defaults during local development, and local subgraph generation. Use `LOCALNET_MANIFEST_PATH` when a consumer needs a non-default manifest path.

The removed on-chain Zap is not a manifest extension: `contracts.zap` and Zap constructor metadata are invalid fields.

## Validation

`packages/sdk/src/manifests/schema.json` documents the canonical shape for generated manifests. The lightweight repository validator checks the example manifests by default:

```sh
pnpm manifests:validate
```

To validate a generated localnet manifest directly, pass the ignored runtime file explicitly:

```sh
pnpm manifests:validate deployments/localnet/latest.json
```

Reference examples live in `deployments/examples/`. They are shape fixtures for SDK, web, CI, and docs work; generated `latest.json` files remain the source of truth for a live local chain.
