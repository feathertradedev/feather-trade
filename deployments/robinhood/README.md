# Robinhood Deployment Manifests

`pnpm robinhood:deploy` writes environment manifests under this directory:

- `testnet/latest.json` for Robinhood Chain Testnet.
- `mainnet/latest.json` for Robinhood Chain mainnet.
- `*/dry-run.json` for dry-runs.

Generated JSON manifests are ignored by git because addresses depend on the deployer nonce and target chain state. Treat `dry-run.json` as a planning artifact and `latest.json` from a broadcast as the live deployment manifest. Operators should not hand-edit generated JSON; re-run dry-run or deploy with the intended environment variables.

## Stable Shape

The stable Wave 2 schema includes `contracts`, `ownership`, `endpoints`,
`tokens`, `chain`, `quoteAssets`, `factoryPreset`, `constructorArgs`, `chainId`,
and `startBlock`.

Robinhood manifests do not include localnet-only `seededPools` or `smoke` fields. The on-chain Zap was removed; `contracts.zap`, top-level Zap metadata, and Zap constructor fields are invalid rather than optional.

## Field Sources

| Manifest field | Source |
| --- | --- |
| `schemaVersion`, `sourceJoeV2Commit`, and `factoryPreset.*` | Generated by the deploy script. |
| `environment` | Generated from the hand-entered `ROBINHOOD_ENV` selector. |
| `chainId`, `startBlock`, and deployed `contracts.*` | Chain-derived from the target RPC and deployment transactions. |
| `deployer`, `ownership.*`, `constructorArgs.initialOwner`, and `constructorArgs.feeRecipient` | Derived from `ROBINHOOD_DEPLOYER_PRIVATE_KEY` or the compatibility fallback. |
| `chain.*` | Generated from the selected Robinhood environment by the shell wrapper. |
| `endpoints.rpcUrl` | Generated from the selected RPC env var or public RPC default. |
| `endpoints.indexerUrl`, `endpoints.apiUrl`, and `endpoints.tokenListUrl` | Hand-entered through optional endpoint env vars; omitted when unset. |
| `tokens.wrappedNative` and `quoteAssets.wrappedNative` | Generated from the canonical Robinhood WETH address, unless `ROBINHOOD_WNATIVE_ADDRESS` is hand-entered. |
| `quoteAssets.extra0` through `quoteAssets.extra3` | Hand-entered through optional env vars. On mainnet, `extra0` defaults to canonical USDG (`0x5fc5360D0400a0Fd4f2af552ADD042D716F1d168`) when unset; other unset slots are zero addresses. |
| Legacy router/factory constructor args | Generated zero addresses for the Wave 1 V2.2-only path. |
| `constructorArgs.quoterFactoryV2_2` and `constructorArgs.quoterRouterV2_2` | Generated from the deployed factory and router addresses. |

Use the manifest with `pnpm robinhood:verify` to verify the deployed contracts on the correct Blockscout explorer.

## Liquidity Composition

Liquidity is added and removed through the canonical `LBRouter`. Native one-sided entry uses `addLiquidityNATIVE` with a compatible bin distribution. A client may optionally guide a user through a separate swap followed by add liquidity, but those are two non-atomic transactions and require a fresh quote and explicit review between signatures. See [Liquidity Composition Decision](../../docs/wave-2/liquidity-composition-decision.md).

## Validation

`packages/sdk/src/manifests/schema.json` documents the canonical shape for generated manifests. The lightweight repository validator checks the example manifests by default:

```sh
pnpm manifests:validate
```

To validate a dry-run or broadcast manifest directly, pass the ignored runtime file explicitly:

```sh
pnpm manifests:validate deployments/robinhood/testnet/latest.json
pnpm manifests:validate deployments/robinhood/mainnet/latest.json
```

Reference examples live in `deployments/examples/`. They are shape fixtures for SDK, web, CI, and docs work; broadcast `latest.json` files remain the source of truth for live deployments.
