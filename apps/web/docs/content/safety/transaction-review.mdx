Before wallet handoff, Feather binds the account, chain, deployment, pool, token order, amounts, settings, recipient, refund destination, approval state, and current market context into one executable review.

## Simulation

The exact target, calldata, and native value are simulated against current chain state. A simulation failure blocks submission and should identify the condition that needs attention.

Changing account, chain, pool, direction, amount, strategy, range, slippage, deadline, approval, balance, active bin, or other bound context invalidates the review.

<Warning title="Inspect the wallet request again">
Simulation reduces avoidable failure but cannot replace wallet review or guarantee confirmation. Reject any wallet request that differs from the application disclosure.
</Warning>
