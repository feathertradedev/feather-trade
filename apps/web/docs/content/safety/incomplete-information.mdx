Feather treats missing, partial, stale, and zero as different states.

- **Loading:** the requested information has not completed.
- **Unavailable:** a value cannot currently be obtained.
- **Partial:** some facts are present, but the complete set is not established.
- **Stale:** information is older than the allowed execution boundary.
- **Zero:** the trusted current value is actually zero.

Read-only presentation can remain visible while dependent actions are disabled. For example, missing USD value does not imply zero value, and an incomplete owned-bin set cannot support a full exit.

<Warning title="Do not work around a safety block">
Refresh or retry the named condition. Switching pages, reconnecting repeatedly, or increasing transaction tolerance does not make untrusted state safe.
</Warning>
